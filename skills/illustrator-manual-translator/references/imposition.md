# Imposition Contract

Read this reference before `impose-ab`, `confirm-ab`, `split-a-b`, or `confirm-a-b`.

## Physical page sequence

1. Treat the configured cover artboard halves as `back_cover` and `front_cover`.
2. Enumerate every remaining artboard in Illustrator order and take its left half before its right half.
3. Keep every physical half-page in that sequence, including unnumbered decoration and existing empty halves.
4. Use printed page-number frames only to detect duplicate or missing labels. Never sort or omit a physical half-page by printed number.
5. Build `front_cover -> physical body pages -> generated padding -> back_cover`.
6. Add completely empty padding only until the total is divisible by four. Put it after the final physical body page and before the back cover.

For six physical body pages, AB must be:

```text
back_cover / front_cover
body_1     / body_6
body_5     / body_2
body_3     / body_4
```

## A/B split

- A contains AB artboards `0, 2, 4, ...`.
- B contains AB artboards `1, 3, 5, ...`.
- A and B must be disjoint and their union must equal every AB artboard exactly once.

## Blocking validation

- Exclude Illustrator guides, hidden items, invisible layers, and non-printing layers from physical-page assignment, but preserve those structural objects in editable AI output; never delete them as a shortcut.
- Block printable objects outside every artboard or crossing the centerline.
- Block duplicate printed page labels and gaps within the discovered printed-label range.
- Require AI/PDF artboard-page count agreement and equal page size.
- Require `MediaBox == CropBox == BleedBox == TrimBox` for zero bleed.
- Require AB and split QA contract version 2 to preserve the expected editable top-level objects and layer count. Reject results generated by a different current imposition runtime hash.
- Encode all JSX file paths as absolute `file://` URIs so non-ASCII paths work through AppleScript.

## Rejection and completion

- A rejected AB may be regenerated while the project is at `waiting_ab_confirmation`; never force a false confirmation to rerun it.
- After A/B confirmation, normal `confirm-a-b` creates the complete delivery package by default.
- If packaging is explicitly declined for this one run, use `confirm-a-b --no-delivery-package`. Store the verified manifest under `work/` and finish at `completed_without_delivery` without writing `delivery/`. This flag is never persisted as a later default.
